# Open Issues

This directory tracks active issues that need resolution.

## Classification

This is a **Workspace**: README.md maintains current status; individual issue files become archived records when moved to `closed/`.

## File Naming Convention

Files are named: `YYYYMMDD-<issue-slug>.md`

Example: `20250130-memory-leak-on-startup.md`

## File Format

Each issue document should include:

1. **Summary**: Brief description of the issue
2. **Symptoms**: How to recognize this issue
3. **Impact**: Severity and affected areas
4. **Status**: Current investigation state
5. **Next Steps**: What needs to happen next

## Lifecycle

1. Create issue file here when discovered
2. Update status as investigation progresses
3. When resolved, move file to `../closed/`
