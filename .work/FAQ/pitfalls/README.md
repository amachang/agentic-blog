# Common Pitfalls

This directory documents mistakes that have been made in this project and how to avoid them.

## Purpose

- Learn from past mistakes
- Prevent recurring issues
- Speed up onboarding for new contributors
- Reduce debugging time

## File Format

Each pitfall document should include:

1. **Problem**: What went wrong
2. **Symptoms**: How to recognize this issue
3. **Root Cause**: Why it happened
4. **Solution**: How to fix it
5. **Prevention**: How to avoid it in the future

## Getting Started

Create your first pitfall document using the format above. As you add files, they will serve as examples for future documentation.
