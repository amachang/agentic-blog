# Sprint History

This directory contains archived sprint files that have been completed.

## File Naming Convention

Files are named: `<timestamp>-<sprint_slug>.md`

Example: `20250103-user-auth-implementation.md`

## Purpose

- Track completed work over time
- Reference past decisions and implementations
- Learn from previous sprints
- Maintain project history

## Usage

Files are automatically moved here when running `/sprint-complete`.
